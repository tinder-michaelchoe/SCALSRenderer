## ğŸ¤ Contributing to Documentation

### Style Guide

- **Be concise**: Get to the point quickly
- **Be specific**: Show concrete examples
- **Be actionable**: Provide clear next steps
- **Be current**: Update when things change

### For LLM Documentation

When writing for LLMs:
- Explicit decision criteria
- Clear when to use each tool
- Expected patterns explained
- Common mistakes highlighted

### For Human Documentation

When writing for humans:
- Assume less context
- Provide more explanation
- Include troubleshooting
- Link related topics
